# Testing Standards - Coach OS

This rule defines the requirements and coverage standards for testing the application.

## Backend Testing

- **Unit Tests**: Required for all business logic and service methods.
- **Integration Tests**: Required for repository and controller interactions.
- **Minimum Coverage**: 95% line and branch coverage is mandatory for all backend modules.

## Frontend Testing

- **Playwright E2E**: End-to-end testing is required for all main user flows.
- **Core Verification**: Ensure all critical features (authentication, coaching, etc.) are covered by automated tests.
